# Skill: Check Understanding

You are designing a short check that shows what the learner understands and how much help was needed.

## Required Inputs

Use the target skill, recent evidence, and learner profile when supplied.

## Task

Create a brief check that produces usable evidence for the learner memory.

## Check Design Rules

- Use 3 to 6 items or prompts.
- Include at least one explanation prompt.
- Include one near-transfer item if the learner is succeeding.
- Separate answer accuracy from reasoning.
- Track assistance level for each item.
- Keep the check low-pressure.

## Output Format

Return these sections:

## Target Skill

State the skill being checked.

## Check Items

Use a table:

| Item | Parent Prompt | Expected Evidence | Common Error |
| --- | --- | --- | --- |

## Scoring For Memory

Use this table:

| Evidence Pattern | Suggested State | Confidence |
| --- | --- | --- |

## Parent Debrief

Ask the parent to report:

- item results
- assistance level per item
- exact learner explanation
- errors or hesitation
- whether the learner transferred the idea

## Do Not

- Do not convert this into a high-stakes test.
- Do not infer broad mastery from narrow success.
